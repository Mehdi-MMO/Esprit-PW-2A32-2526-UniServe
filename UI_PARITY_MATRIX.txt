UniServe INTEG — UI parity audit (modules)
============================================
Date: 2026-05-13
Scope: Codebase-only review. External “reference” trees (RDV_MODULE, DEMANDE_MODULE,
DOCAC_MODULE, etc.) are NOT in this repository (.gitignore / local copies). Rows marked
“Externe” cannot be YES/NO without dropping those folders into THEM MODULES and diffing.

Legend
------
  Aligné UniServe   = Uses the same shell as most INTEG pages (us-page-header, us-kicker,
                      us-section-card, shared alerts).
  Partiel           = Same shell but different density (no stats grid, no hero, table-only).
  Écart ref “riche” = Compared to the *student RDV hub* style (hero, KPI row, map, card grid,
                      export): this surface is still “classic portal” (header + table/form).
  n/a               = Not a CRUD module UI (auth, API, chat JSON, etc.).

Transversal (interne INTEG)
---------------------------
  • Seul le rendez-vous étudiant charge `rendezvous-hub.css` (gradient hero, carte décorative).
  • Certifications vit hors `View/frontoffice/*` → structure HTML/CSS différente (breadcrumb
    maison, `card shadow-sm` vs `us-section-card`).
  • Plusieurs vues utilisent `us-page-header` avec `mb-4` vs sans, ou `gap-2` vs `gap-3` : détail
    cosmétique, pas fonctionnel.
  • Export PDF “style ref” : RDV étudiant (exportMesPrint) + RDV admin (exportPrint) ; autres
    modules sans équivalent print.

Comment utiliser ce fichier avec les modules équipe
----------------------------------------------------
  1. Copier chaque arbre module (ex. RDV_MODULE, DEMANDE_MODULE) dans THEM MODULES/ (voir
     THEM MODULES/README.txt).
  2. Pour chaque ligne “Inconnu / Écart”, ouvrir les deux UIs côte à côte et ajouter une colonne
     “Captures / priorités” (P0 hero, P1 stats, P2 cards…).
  3. Éviter de tout homogénéiser sur le gabarit RDV : garder `us-page-header` + `us-section-card`
     partout, et n’ajouter hero/map/cards que là où le produit l’exige.
